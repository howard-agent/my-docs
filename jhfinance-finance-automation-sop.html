一、系统架构总览 ARCH

ℹ️ 适用项目：某跨境电商公司内部财务看板。环境：WSL2 (Ubuntu) + VS Code Remote + GitHub Pages。 
raw-data/          ← 原始 Excel 财务文件（手动上传）
analysis/
  ├── generate_reports.py      ← 读取 Excel，生成财务三表
  ├── generate_html_report.py  ← 生成 HTML 看板 + 发布到 GitHub Pages
  ├── lingxing_api.py          ← 领星 ERP API 客户端（拉订单/库存）
  ├── costs_config.json        ← 月度固定成本配置（人工/租金/订阅/仓储）
  ├── daily_update.sh          ← 每日自动更新脚本（crontab 调用）
  ├── .lx_data_cache.json      ← 领星数据本地缓存（自动生成，勿手动改）
  └── .lx_token_cache.json     ← 领星 API Token 缓存（自动生成）
monthly-reports/               ← 生成的 HTML 报告本地存档
JHFinance-public/              ← GitHub Pages 仓库（git submodule 风格）

二、数据来源说明 DATA

2.1 手工 Excel 数据（历史财务）

放在 raw-data/ 下，文件格式经历了几次变迁：

时期 文件 内容
2025年1-3月 A1_xxx.xlsx RMB 收支，多 sheet
2025年4-12月 A2_xxx.xlsx / 锦和2025年4-5月xxx.xlsx RMB 收支
2025年（美元月份） A3资产表格.xlsx USD 报表，需汇率换算
2026年1-2月 深圳市锦和科技有限公司202x年x月资产表格.xlsx 独立 RMB 三表（含利润/资产负债/现金流 sheet）
2026年3月起 锦和记账明细表20260330.xlsx 流水明细，实时更新
⚠️ 更新方式：直接替换 raw-data/ 下对应文件，文件名有变化时需更新 generate_reports.py 里的 STANDALONE_2026 字典。

2.2 领星 ERP API 数据（自动拉取）

每天北京时间 08:00 自动拉取，覆盖当月 + 上月数据。

  • 1亚马逊订单：/erp/sc/data/mws/orders
  • 2多平台订单（Walmart / TikTok / Shopify / Shopee / AliExpress 等）：/pb/mp/order/v2/list
  • 3本地库存产品目录：/erp/sc/data/local_inventory/productList
  • 4店铺列表：/erp/sc/data/seller/lists

六、每日自动更新流程 AUTO

自动运行（已配置 crontab）

✅ crontab 规则：0 0 * * * → 北京时间 08:00 自动执行 
daily_update.sh
  └─ 1. lingxing_api.py pull     （拉领星数据 → .lx_data_cache.json）
  └─ 2. generate_reports.py      （读 Excel + 缓存 → 财务三表）
  └─ 3. generate_html_report.py  （生成 HTML → 推送 GitHub Pages）

日志查看

tail -f /home/howard/wsl-workspace/JHFinance/daily_update.log

手动触发（任意时刻刷新）

cd /home/howard/wsl-workspace/JHFinance
bash analysis/daily_update.sh

或分步单独运行：

python3 analysis/lingxing_api.py pull         # 只拉数据
python3 analysis/generate_html_report.py      # 只刷新看板

八、新增财务数据操作步骤 OPS

场景 A：更新当月流水（月中 / 月末）

▶

1. 替换 raw-data/锦和记账明细表YYYYMMDD.xlsx（更新文件名日期）

2. 更新 generate_reports.py 中 load_journal() 函数里的文件名

3. 运行：

bash analysis/daily_update.sh

场景 B：新月份有独立三表文件（如 2026-04 月）

▶

1. 将新文件放入 raw-data/

2. 在 generate_reports.py 的 STANDALONE_2026 字典中追加：

'2026-04': {
    'file': '深圳市锦和科技有限公司2026年4月资产表格.xlsx',
    'profit': '4月利润表',
    'balance': '4月资产负债表',
    'cashflow': '2026年4月现金流量表'
}

3. 确认 sheet 名称与实际文件一致

4. 运行：

bash analysis/daily_update.sh

场景 C：成本变化（加减员工 / 调整租金等）

▶

1. 编辑 analysis/costs_config.json，新增对应月份的配置

2. 重新运行看板生成：

python3 analysis/generate_html_report.py

十、常见问题排查 DEBUG

问题 原因 解决
看板打不开 / GitHub 404 Jekyll 处理中文文件名失败 确认 JHFinance-public/.nojekyll 文件存在
领星返回 sign error (2001006) 签名参数有误 POST 请求需把 body 参数也纳入签名计算
多平台订单为 0 API 分页 total 字段为字符串 已修复：int(data.get('total') or 0)
收入数字偏小（只有 USD 部分） 多平台 MXN/AED 未换算 检查 costs_config.json 中对应汇率是否配置
pip 安装失败（venv shebang 错误） macOS venv 带了 Mac 路径的 shebang pip3 install --user --break-system-packages <包名>
git push 失败（SSH 权限） WSL 无 SSH key 配置 改用 HTTPS：git remote set-url origin https://github.com/...
