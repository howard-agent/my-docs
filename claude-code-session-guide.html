一、Session 核心机制 CORE

Session 是 Claude Code 的对话历史记录，以 JSONL 格式存储在本地磁盘。每个 session 包含你的 prompt、Claude 的每次工具调用和响应。

绑定规则
绑定项目绝对路径
只能在创建时的目录 --resume
路径编码
/ 替换为 -
/home/user/proj → -home-user-proj
文件格式
JSONL append-only
<session-uuid>.jsonl
Local vs Web
完全独立
Local 存本机，Web 存 Anthropic 云端
⚠️ Session 不能移动到其他文件夹——绑定创建时的目录，无法迁移。

存储路径结构

~/.claude/
├── projects/
│   └── [路径编码]/
│       ├── <session-id>.jsonl          # 完整对话记录
│       ├── <session-id>/subagents/     # 子代理记录
│       ├── sessions-index.json         # session 索引
│       └── memory/                     # auto memory
├── history.jsonl                       # 全局输入日志
└── settings.json                       # 全局设置

Session 类型对比

类型 存储位置 可否跨设备 管理方式
Local (CLI) 本机 ~/.claude/projects/ 需手动迁移 claude --resume
Local (VSCode) 同上，索引可能不同步 同上 VSCode Local tab
Web Anthropic 云端 任意浏览器 claude.ai Code tab
Remote Control 云端镜像 镜像本地 session 自动生成，可删

Session 命名

在 Claude Code session 内输入命令，建议用统一前缀编号：A 系列（Amazon）、B 系列（白云站）、P 系列（个人项目）、C 系列（儿童项目）等。

/name A1_项目名_描述

二、三种界面的关系 IMPORTANT

终端 CLI
最可靠
claude --resume · 显示当前目录所有 Local session
VSCode 面板 Local
索引经常不全
显示 Local session，但不同步
Web / VSCode Web
显示 Web session
Remote Control 镜像也在此
💡
核心原则：永远只用终端 + tmux 跑 Claude Code，不用 VSCode 面板。VSCode 面板只用来看代码和编辑文件。

Remote Control 说明

  • 1Remote Control 让你从手机/网页端控制本地 CLI session
  • 2它在 claude.ai 网页上生成一个镜像 session
  • 3镜像 session 结束后不能再对话（显示 "No response requested"）
  • 4删除镜像不影响本地 session——真正的对话在本地机器
  • 5Pro plan 可能没有 /remote-control 命令（版本相关）

三、常见错误与解决 DEBUG

No conversation found with session ID: xxx

▶

Session 元数据被索引但 JSONL 文件未写入成功。常见于网络中断。

不可恢复，定期清理空文件（97字节为空 session）：
find ~/.claude/projects/ -name "*.jsonl" -size 97c -delete

VSCode Local tab 显示 "No sessions yet"

▶

VSCode 没有打开项目文件夹。需要 File → Open Folder 打开对应目录后即可显示。

VSCode 快捷键冲突

▶

Mac 远程 SSH 时 Ctrl+R 等快捷键被 VSCode 截获，无法传入终端。

使用本地键盘或直接通过命令行 claude --resume 管理 session。

"Interrupted · What should Claude do instead?"

▶

Subagent 权限确认超时导致中断。

输入"继续执行"恢复，或在 settings.json 中预配置权限避免每次确认。

Auto-update failed

▶

自动更新权限不足，手动更新即可：

sudo npm i -g @anthropic-ai/claude-code

四、跨设备 Session 迁移实操 MIGRATE

以 MacBook → 台式机 WSL 为例，Session 与代码分开迁移。

# MacBook: 打包 session（目录名以-开头必须加--）
cd ~/.claude/projects
tar -czf ~/claude-sessions-backup.tar.gz -- -Users-macuser*

# MacBook: 打包代码
tar -czf ~/project-code.tar.gz -C ~/Documents ProjectName

# SCP 到台式机
scp ~/claude-sessions-backup.tar.gz ~/project-code.tar.gz user@[内网IP]:~/

# 台式机: 解压并重命名路径编码
tar -xzf ~/project-code.tar.gz -C ~/wsl-workspace
mkdir -p /tmp/claude-migrate
tar -xzf ~/claude-sessions-backup.tar.gz -C /tmp/claude-migrate
cp -r /tmp/claude-migrate/-Users-macuser-Documents-ProjectName \
  ~/.claude/projects/-home-user-wsl-workspace-ProjectName
rm -rf /tmp/claude-migrate

# 验证
cd ~/wsl-workspace/ProjectName && claude --resume
✅ 迁移后 claude --resume 能看到历史 session 即为成功。若不可见，检查路径编码是否与当前目录匹配。

五、Session 清理 SOP CLEAN

# 1. 删除空 session
find ~/.claude/projects/ -name "*.jsonl" -size 97c -delete

# 2. 删除 macOS 元数据
find ~/.claude/projects/ -name "._*" -delete

# 3. 查看各项目 session 数量
for dir in ~/.claude/projects/*/; do
  count=$(find "$dir" -maxdepth 1 -name "*.jsonl" | wc -l)
  echo "  $(basename $dir): $count 个session"
done

# 4. 按大小列出 session，对照 resume 标题定位删除
ls -l ~/.claude/projects/-home-user-wsl-workspace/*.jsonl | awk '{print $5, $9}' | sort -n
# 然后 rm 对应文件
