🔌 核心大模型通道接入指南 3 Providers

OpenClaw 3.8 的 Provider 校验机制极其严格（强迫症级别），不同平台接入方式有所不同，以下逐一记录踩坑与解决方案。

AIHubMix — 官方插件自动化接入（最省心）

▶

AIHubMix 提供专用认证插件，可自动同步分类所有模型（涵盖 Claude、Gemini、OpenAI 等）。

① 安装插件：

openclaw plugins install @akaknele/aihubmix-auth

② 重启容器（宿主机执行）：

sudo docker restart openclaw

③ 交互式认证：

openclaw models auth login --provider aihubmix --method api-key --set-default
按提示输入 API Key，插件会自动配置路由和 Header，无需手动填写。

OpenRouter — 系统原生免驱接入

▶

OpenRouter 在 3.8 版本中作为 Built-in 原生支持，不需要使用 plugin login 命令。

配置方式：直接修改配置表，或通过命令写入 Key：

openclaw config set auth.openrouter "[密钥]"

调用时在模型前加前缀即可，例如：

openrouter/anthropic/claude-3-opus

VectorEngine — 严格校验踩坑 & 复合 JSON 注入方案

▶

VectorEngine 作为 OpenAI 兼容接口，在尝试用 openclaw config set 逐行写入时，遭遇实时完整性校验拦截——缺少 baseUrl、缺少 models 数组等都会直接报错阻断。

⚠️ 逐字段写入会被 OpenClaw 校验器在中间态拦截报错，必须一次性注入完整对象。

最终解决方案（复合 JSON 注入）：

openclaw config set models.providers.vectorengine '{
  "api": "openai-completions",
  "baseUrl": "https://api.vectorengine.ai/v1",
  "apiKey": "[密钥]",
  "models": [
    { "id": "deepseek-v3", "name": "DeepSeek V3" },
    { "id": "minimax-abab6.5-chat", "name": "MiniMax" }
  ]
}'
将完整对象（含 api 类型、URL、Key、Models 数组）一次性写入，绕过中间态校验。

🧩 智能体 (Agent) 的配置与指派 已验证

Provider 管道接通后，需要创建具体 Agent 并指派对应的 Provider 和 Model。

基础命令规范

openclaw config set agents.<角色名>.provider "<对应Provider名称>"
openclaw config set agents.<角色名>.model "<对应模型ID>"

实战举例：配置 worker 分身

openclaw config set agents.worker.provider "vectorengine"
openclaw config set agents.worker.model "deepseek-v3"
✅ 配置完成后重启容器，即可在飞书内通过 /worker <指令> 调用该分身。

🌐 网络与外网 Web UI 访问排查手册 多层阻击

在尝试访问 http://[内网IP]:18789/?token=... 的 Web GUI 时遭遇各类网络问题，以下按层次记录排查经验。

宽带与 DDNS 限制 — ERR_CONNECTION_CLOSED

▶
现象使用腾讯云 DDNS 域名访问报 ERR_CONNECTION_CLOSED
根因家宽无公网 IPv4，仅配置了 IPv6 DDNS
外网设备（手机移动网络或部分公司 Wi-Fi）不支持 IPv6 时，DDNS 域名解析直接失效，连接被拒。

Tailscale + v2rayN 冲突 — 虚拟局域网流量被劫持

▶

为实现"随处访问"，启用了 NAS 上的 Tailscale 内网穿透。

冲突点Mac 端 v2rayN 代理规则劫持/丢弃 Tailscale 的 100.x.x.x 虚拟局域网流量
💡 解决方案：在 v2rayN 中开启 Enable Tun（虚拟网卡模式），让系统级路由正确接管科学上网和 Tailscale 流量，两者互不干扰。

终极 503 报错 — HTTP ERROR 503

▶

网络打通后，通过 Tailscale IP 访问依然报 503，存在两个根因：

原因 1容器内 Gateway 默认只监听 127.0.0.1，拒绝外部访问
修复：在 Docker 环境变量中补充 HOST=0.0.0.0，让 Gateway 监听所有网卡。
原因 2频繁修改配置导致 Gateway 进程假死
openclaw gateway stop
openclaw gateway start
⚠️ Web UI 持续 503 期间，果断转向纯 CLI 命令行完成全部配置，避免阻塞进度。
