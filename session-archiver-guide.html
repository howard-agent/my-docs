🤖 它是什么 Agent

session-archiver 是一个专门用于归档 Claude Code 工作记录的 subagent。每次完成一个工作 session（调试、配置、功能开发等），由它提取关键信息，生成结构化中文 Markdown 文档，自动投递到 inbox/，触发发布流水线。

Agent 名称
session-archiver
配置文件：~/.claude/agents/session-archiver.md
模型
Claude Sonnet
工具权限：Glob / Grep / Read / Edit / Write
输出目录
inbox/
命名格式：topic_YYYYMMDD.md
文档语言
中文
代码块 / 路径保留英文

📞 如何调用 调用方式

有三种方式触发 session-archiver：

方式一：自然语言（最常用）

在 session 快结束时，直接对 Claude 说：

帮我归档这个 session
archive this session
总结今天的工作并归档

方式二：斜杠命令

/session-archiver

方式三：在消息中明确引用

用 session-archiver agent 把今天的网络配置工作归档一下
触发时机建议：完成一个独立功能 / 解决一个 bug / 配置完成某个系统后，即可触发。不必等到整个项目结束。

📄 输出格式 结构

每次归档生成的 MD 文件固定包含五个章节：

章节内容
背景为什么做这件事？问题起因或业务需求
关键操作按时间顺序的步骤，含 shell 命令和配置变更
结果成功 / 失败 / 部分完成，以及验证方式
遗留问题未解决事项，无则写「无」
相关文件所有创建、修改、删除的文件路径

文件命名示例：

tailscale_network_20260401.md
my_docs_automation_20260402.md
jhfinance_erp_sync_20260330.md
隐私保护：agent 会自动将密码、IP、Token 等敏感信息替换为 [已脱敏]，再由 watcher 做第二轮脱敏。

🔄 追加模式 同话题更新

如果同一话题（如 tailscale_network）已有对应 HTML，新的归档文件会自动追加为新章节，而不是创建新 HTML。

Day 1：tailscale_network_20260401.md 投入 inbox/

无对应 HTML → 新建 tailscale-network.html，加入 index

Day 2：tailscale_network_20260402.md 投入 inbox/

tailscale-network.html 已存在 → 追加新日期章节，index 卡片日期更新并移到首位

结果

一个话题始终只有一个 HTML，内容按日期累积

判断逻辑：文件名符合 topic_YYYYMMDD.md 格式，且 topic.html（下划线转连字符）已存在，则触发追加模式。
