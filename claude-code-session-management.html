📦 02 · Session 核心机制

🔑 核心规则

Session 绑定项目绝对路径。在 /home/[user]/wsl-workspace/Amazon-agents 下创建的session，只能在这个目录下 claude --resume 找到。

Session存储路径编码规则

路径中的 / 被替换为 -：

/home/[user]/wsl-workspace/Amazon-agents
→ ~/.claude/projects/-home-[user]-wsl-workspace-Amazon-agents/

Session类型对比

类型存储位置可否跨设备管理方式
Local本机 ~/.claude/projects/需手动迁移CLI / VSCode Local tab
WebAnthropic云端任意浏览器claude.ai Code tab

关键命令

命令作用
claude在当前目录新建session
claude --continue继续当前目录最近的session
claude --resume选择器浏览所有session
claude --resume <session-id>恢复指定session
claude -p "任务"Headless模式，执行完退出（适合自动化）
/clearsession内清空上下文，开始新对话

2
MacBook本地跑临时session
直接在MacBook项目目录：cd ~/Documents/Amazon-agents && claude
Session保存在MacBook本地的 ~/.claude/projects/
产出的代码改动通过git同步：git add . && git commit -m "改动说明" && git push
台式机拉取：cd ~/wsl-workspace/Amazon-agents && git pull
💡 如果后来想把session转到远端

Session本身无法远程"转移"。但代码改动可以通过git同步。最佳做法：把关键决策写入CLAUDE.md，然后在台式机上开新session继续。

3
多个session并行
每个项目在自己的目录下有独立session，互不影响
用tmux多窗口管理：tmux new -s amazon / tmux new -s kidsgame
切换：tmux switch -t amazon 或 Ctrl+B 然后 s 选择
⚡ Pro Plan注意

同时活跃的session共享rate limit。建议同时不超过2个活跃session。一个在跑长任务时可以开另一个做快速查询。

4
手机端紧急任务
📱 推荐方案：SSH到台式机

手机装SSH客户端（Termius / iSH / 系统终端），连接台式机Tailscale IP，在tmux里操作。代码和session都留在台式机上。

手机SSH：ssh user@[YOUR_TAILSCALE_IP]
恢复tmux：tmux attach -t work
或者开新session：cca（进入Amazon-agents并启动claude）
🌐 备选：claude.ai网页端

手机浏览器打开 claude.ai → Code tab → 可以派发Web session。但Web session和Local session独立，不互通。适合完全独立的小任务。

5
定期备份Mac session到台式机
# MacBook上打包
cd ~/.claude/projects
tar -czf ~/claude-sessions-$(date +%Y%m%d).tar.gz -- -Users-[macuser]*

# 传到台式机
scp ~/claude-sessions-$(date +%Y%m%d).tar.gz user@[YOUR_TAILSCALE_IP]:~/backups/

# 台式机上保存（纯存档，不做路径映射）
mkdir -p ~/backups
# 文件已在~/backups下，按日期归档

🔧 06 · 故障排查

❌ "No conversation found with session ID: xxx"

原因：Session元数据被索引但JSONL文件未写入成功。常见于手机远程派任务网络中断时。

解决：这些session不可恢复。在VSCode Local tab hover删除，或用命令行删除小于1KB的JSONL文件：

find ~/.claude/projects/ -name "*.jsonl" -size -100c -delete
⚠️ VSCode Local tab显示 "No sessions yet"

原因：VSCode没有打开项目文件夹（显示欢迎页）。Session绑定目录，需要先 File → Open Folder 打开项目。

⚠️ Web session无法在VSCode Local tab管理

Web session存在Anthropic云端，只能在 claude.ai 网页或VSCode的 Web tab 中管理。与Local session完全独立。

💡 macOS垃圾文件（.DS_Store / ._*）

从Mac传文件到Linux时会带入。清理方法：

find . -name '.DS_Store' -delete
find . -name '._*' -delete

项目 .gitignore 中已配置忽略。

💡 Session跨机器迁移

打包 ~/.claude/projects/，scp到目标机器，重命名目录以匹配新的绝对路径编码。但官方建议：与其搬session，不如把上下文写入 CLAUDE.md，在新机器开新session。
